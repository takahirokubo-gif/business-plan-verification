## 7. KPIツリー（S4）の期待構造

財務モデルの数式チェーン（inpシート→PLシート）から、以下のエッジが再現されること：

```
売上収益
├─ EC売上（=会員数×購入回数×AOV）
│   ├─ アクティブ会員数（=前期会員数×リピート率＋新規獲得会員数）
│   │   ├─ リピート率 ★
│   │   └─ 新規獲得会員数 ★候補
│   ├─ 年間平均購入回数
│   └─ 平均注文単価（AOV）
├─ 卸売上（=店舗数×店舗あたり年間出荷額）
│   ├─ 卸取扱店舗数 ★候補
│   └─ 店舗あたり年間出荷額
└─ その他売上（百貨店・海外）
売上原価 ─ 売上原価率 ★候補（OEM値上げリスク）
広告宣伝費 ─ 広告宣伝費率 ★候補（CPA上昇リスク）
```

- ★（リスクドライバー）は最低2つ。候補：リピート率・新規獲得・原価率・店舗数・広告費率
  （いずれもDDの指摘と紐づく。候補外への★は要人手レビュー）
- DD由来ノード（モデル外・定性バッジ）の候補：OEM委託先依存度・卸チェーン集中度
- 各ノードに根拠（シート・行、数式のどの部分から親子関係を判定したか）が付くこと

## 8. シナリオ（S5）のルーブリック（生成タスクのため一意解なし）

AI推奨3類型が生成され、それぞれ標準5部構成が埋まっていること：

| 類型 | 期待される題材（いずれか） | 引用されるべき事実 |

| トップライン | リピート率低下／大口卸チェーン取引縮小 | リピート率76%・上位3チェーン45%（最大手21%） |

- 「AI推定・モデル再計算なし」バッジ必須。判定ラベル（問題なし/問題あり等）は出さない
- 変化幅には根拠（DDの事実・外部データ）が紐づくこと
