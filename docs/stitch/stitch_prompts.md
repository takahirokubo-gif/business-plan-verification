# Stitch プロンプト集：案件詳細画面のUIUX再構成

新しいタブ構成（Overview ＋ 4つの確認・修正タブ）を Stitch で検証するためのプロンプト集。

## 使い方

1. Stitch で新規プロジェクトを作成（Web / Desktop）
2. 最初に **「0. 共通デザインシステム」** をテーマ指示として貼る（または各画面プロンプトの末尾に毎回付ける）
3. 画面ごとに 1〜5 のプロンプトを順に投入する（1画面ずつ生成するのが精度が高い）
4. 参照画像として同フォルダの2ファイルのスクリーンショットを添付する
   - `style_guide.html` … トンマナ（色・タイポ・コンポーネント）の参照
   - `overview_mock.html` … Overview画面の構成・情報密度の参照

## 0. 共通デザインシステム（毎回添える）

```
デザインシステム（全画面共通・添付のスタイルガイド画像に厳密に従うこと）:
- 銀行の融資審査部門向けB2B業務システム。信頼感・精密さ・落ち着きが最優先。デスクトップ専用（1440px想定）
- 配色: 背景 #F9F9FF、カードは白 #FFFFFF に 1px ボーダー #E2E2E8。影は一切使わない（フラット）
- プライマリ: Trust Blue #1A4F8B(ボタン・アクティブタブ)、濃紺 #00386C(ホバー)。アクセントは最小限
- フォント: 日本語 Noto Sans JP、数値は Inter(tabular-nums で桁を縦に揃える)。本文13-14px、見出しは太字で階層を作る
- 角丸は4px統一。テーブルの行は角丸なしで高密度に
- ステータスバッジ: 薄い背景色＋同系のテキスト色＋1pxボーダー、角丸8px。
  確定=緑(#ECFDF3/#027A48)、未確定・要確認=アンバー(#FFFAEB/#B54708)、
  エラー・不整合=赤(#FFDAD6/#93000A)、AI推定=紫(#F4F0FF/#6941C6)、中立=グレー
- レイアウト: 左に固定サイドバー260px(薄グレー背景、アクティブ項目は左4pxのTrust Blueアクセント＋薄青背景)。
  メイン領域は最大幅1280px、外側余白24px、カード間16px
- 画面上部: パンくず → 案件名(22px太字)＋審査ステータスバッジ → タブバー(下線型。アクティブはTrust Blueの太字＋2px下線)
- タブバー構成(全画面共通): 概要 | 事業概要 | KPIツリー | 財務ダイジェスト | ストレス仮説
- 重要な業務ルール: AIが推定した値と資料から抽出した事実は必ず視覚的に区別する(AI推定=紫バッジ)。
  未確定の値は薄い黄色の背景で示す。すべての値に「根拠」(参照ファイル・箇所・抽出論理)を辿る導線がある
```

## 1. 概要（Overview）タブ

```
銀行の融資審査官向け「LBO案件検証」アプリの案件詳細画面。タブ「概要」がアクティブ。
案件全体を1画面で俯瞰する読み取り専用のダッシュボード。添付のOverviewモック画像の構成を再現すること。
上から順に4つのセクションカードを縦に積む。各カードのヘッダーは「▼ セクション名」(太字)＋
右端に「編集」テキストリンク(対応するタブへ遷移する想定)＋未確定件数バッジ(例:「未確定 3」アンバー)。

1) 案件基本情報・事業要約:
   左に基本情報のキーバリュー表(案件名、スキーム: LBO、対象会社、EV、エクイティ/デット構成、スポンサー)、
   右に「事業要約」の段落テキスト3行(自動車完成車メーカーおよびTier1/Tier2部品メーカーの工場に対する
   製造ラインオペレーター派遣ビジネス、という内容)

2) 重要KPIとその構造(ツリー):
   数式ツリーをインデント付きリストで表示。
   「売上高 = 稼働人数 × 月間平均稼働時間 × 派遣単価(時間あたり)」
   「★ 稼働人数 = (前期末登録者数 + 新規採用数 − 退職数) × 稼働率」(★=最重要KPI、Trust Blueで強調)
   「売上原価 = 稼働人数 × 月間平均稼働時間 × 派遣スタッフ時給」
   「販管費 = 採用費(新規採用数 × 採用単価/CPA) + 本社固定費」
   「EBITDA = 売上高 − 売上原価 − 販管費 + 減価償却費」

3) 財務情報(単位: 百万円):
   横スクロール可能な高密度テーブル。列は3グループのヘッダー帯:
   「確定財務」(FY24 FY25 FY26)/「ベースケース」(FY27〜FY31)/「スポンサーケース」(FY27〜FY31)。
   行は左端に縦書き風のグループラベル「PL」「重要KPI」「BS」「CF」を付け、
   PL: 売上/売上総利益/同率/営業利益/同率/経常利益/同率/当期利益/EBITDA、
   重要KPI: 稼働率/新規採用数/派遣単価/採用CPA、
   BS: 現預金/のれん/純資産/総資産、
   CF: 営業CF/投資CF/財務CF/FCF。
   数値はInterのtabular-numsで右揃え。一部セルは薄黄色(未確定)、1セルに⚠アイコン(資料間不整合)

4) ストレス仮説とその根拠データ:
   シナリオA/B/Cの3枚のサブカード。各カードはタイトル行
   (例:「シナリオA: 自動車減産に伴う『急激な稼働率低下』(トップライン・ショック)」)＋
   ラベル付き箇条書き:【KPIとリスク】【ストレスと根拠】【インパクト】【保全策・構造】【Q&A】。
   各カードに紫の「AI推定」バッジと採用状態を示すトグル
```

## 2. 事業概要タブ（確認・修正）

```
同じアプリ・同じ案件詳細画面で、タブ「事業概要」がアクティブ。
案件基本情報と事業要約を「AIの提案 → 人がレビュー → 確定」の流れで確認・修正する画面。
```

## 3. KPIツリータブ（確認・修正）

```
同じアプリ・同じ案件詳細画面で、タブ「KPIツリー」がアクティブ。
Excelの数式から解析したKPI構造ツリーを確認・修正する画面。

- 左2/3: 「KPI構造ツリー」カード。ルート「EBITDA」から展開するインデント型ツリー。
  各ノードは 名称＋数式(例: 稼働人数 = (前期末登録者数 + 新規採用数 − 退職数) × 稼働率)。
  ノード右端に: ★トグル(最重要KPI指定、Trust Blueの星)、
  シナリオ使用チップ(小さな色付きチップ「A」「B」…このKPIがどのストレスシナリオで使われているか)、
  「⋯」メニュー(数式を編集/ノードを分割/削除)。
  1つのノードが選択状態(薄青背景＋左3px Trust Blueボーダー)
- 右1/3: 選択ノードの詳細パネル。数式の編集入力欄、
  根拠3点セット(①参照ファイル: BankBaseモデル.xlsx ②箇所: シートModel セルD25 ③解析の論理)、
  「このKPIを使用中のシナリオ: シナリオA」のリンクチップ
- ツリーカードの上部に注記バー(薄グレー):
  「KPI構造はExcel数式の構文解析結果です。確定したKPI構造はシナリオ検討の入力になります」
- 下部フッター: 「KPI構造を確定」プライマリボタン(右端)、左に「確定するとシナリオタブが解放されます」の補足
```

## 4. 財務ダイジェストタブ（確認・修正）

```
同じアプリ・同じ案件詳細画面で、タブ「財務ダイジェスト」がアクティブ。
資料から拾い上げた財務数値を1セルずつ確認・確定するグリッド画面。Overviewの財務情報テーブルと同じ構成。

- メイン: 財務テーブル(単位: 百万円)。列グループ「確定財務 FY24-26」「ベースケース FY27-31」
  「スポンサーケース FY27-31」、行グループ「PL」「重要KPI」「BS」「CF」(行項目はOverviewと同じ)。
  セル状態: 白=確定済、薄黄色=未確定、赤枠＋⚠=資料間不整合。1つのセルが選択中(Trust Blue 2px枠)
- テーブル上部にツールバー: フィルタトグル「未確定のみ表示」、凡例(薄黄=未確定 ●=未確定あり ⚠=不整合)、
  右端に「表示中の12件を一括確定」セカンダリボタン
- 右側: 選択セルの「根拠」スライドパネル。値(1,240)と根拠3点セット
  (①参照ファイル: 決算書_FY25.pdf ②箇所: P.3 損益計算書 ③抽出の論理)、
  不整合がある場合の比較表示(資料A: 1,240 / 資料B: 1,255)、
  「この値で確定」プライマリボタン、「値を修正して確定」入力欄
- ヘッダー下に進捗: 「確定 84/96 セル」プログレスバー＋「不整合 1件」赤バッジ
```

## 5. ストレス仮説タブ（確認・修正）

```
同じアプリ・同じ案件詳細画面で、タブ「ストレス仮説」がアクティブ。
AIが提案したストレスシナリオを確認・編集し、採用するものを選ぶ画面。

- メイン: シナリオカードのリスト(A/B/C)。各カードのヘッダー行:
  色付きのシナリオキー(A=青系, B=アンバー系, C=赤系の丸チップ)、
  タイトル(例:「シナリオA: 自動車減産に伴う『急激な稼働率低下』(トップライン・ショック)」)、
  紫「AI推定・モデル再計算なし」バッジ、右端に「採用」トグルスイッチ。
  カードA=展開状態: 5つのラベル付きフィールドがそれぞれ編集可能なテキスト行として並ぶ
  【KPIとリスク】【ストレスと根拠】【インパクト】【保全策・構造】【Q&A】。
  各フィールド右端に鉛筆アイコン。「根拠: 過去の経済ショック時の業界平均下落幅」の部分は
  下線リンク(クリックで根拠パネル)。関連KPIチップ(稼働率、新規採用数)がフィールド下に並ぶ。
  カードB/C=折りたたみ状態(タイトル行のみ)
- リスト下部: 破線ボーダーの「＋ 自分の仮説シナリオを追加」カード
  (例文プレースホルダ:「大口派遣先(構成比28%)が契約終了したら」)
- 右1/3: AIチャットパネル(常設)。ヘッダー「シナリオ検討アシスタント」、
  吹き出し2-3個の会話、下部に入力欄。AIの発言には紫の小さなAIアイコン
- 下部フッター: 「採用シナリオを確定してエクスポートへ」プライマリボタン(右端)、
  左に「採用中: 2/3 シナリオ」カウンタ
```
